# [2580 스도쿠](https://www.acmicpc.net/problem/2580)

## 1~3번째 시도

재귀를 활용한 완전탐색으로 풀어보자

빈 칸이 정확히 몇 개 나올진 모르겠지만 시간을 절약하기 위해 빈 칸에 숫자를 넣기 전에 가로줄, 세로줄, 3x3 칸에 중복된 숫자가 있는지 확인한 뒤 넣도록 하자

```cpp
#include <iostream>
#include <vector>

using namespace std;

int board[9][9];

struct Point {
    int y;
    int x;
};

bool checkRow(int y, int num) {
    for(int x=0;x<9;x++) {
        if(board[y][x] == num) return false;
    }

    return true;
}

bool checkCol(int x, int num) {
    for(int y=0;y<9;y++) {
        if(board[y][x] == num) return false;
    }
    return true;
}

bool checkBox(int y, int x, int num) {
    int sy = (y/3)*3;
    int sx = (x/3)*3;

    int ny, nx;
    for(int dy=0;dy<3;dy++) {
        for(int dx=0;dx<3;dx++) {
            ny = sy+dy;
            nx = sx+dx;

            if(board[ny][nx] == num) {
                return false;
            }
        }
    }

    return true;
}

void solve(vector<Point>& v) {
    if(v.size() == 0) {
        for(int y=0;y<9;y++){
            for(int x=0;x<9;x++) {
                cout<<board[y][x]<<" ";
            }
            cout<<endl;
        }
        return;
    }
    
    Point blank = v.back();
    v.pop_back();
    int y = blank.y;
    int x = blank.x;

    for(int num=1;num<=9;num++) {
        if(checkBox(y, x, num) && checkCol(x, num) && checkRow(y, num)) {
            board[y][x] = num;
            solve(v);
        }        
    }
}

int main() {
    vector<Point> v;

    for(int y=0;y<9;y++) {
        for(int x=0;x<9;x++) {
            cin>>board[y][x];
            if(board[y][x] == 0) {
                v.push_back({y, x}); 
            }
        }
    }
    
    solve(v);
    return 0;
}
```

## 5번째 시도

스도쿠 문제 푸는 사이트에서 예제를 하나 찾아서 돌려봤다

```
0 9 8 0 5 0 6 7 2
2 0 6 0 0 9 0 0 0
5 0 0 2 7 0 0 0 0
3 0 7 0 9 2 0 0 8
4 0 0 0 0 0 0 0 6
9 0 0 4 6 0 2 0 7
0 0 0 0 8 5 0 0 4
0 0 0 9 0 0 7 0 5
6 5 2 0 4 0 3 8 0
```

이 케이스에서 문제를 찾았다

(1,3)에 1을 대입했을 때, (1,4)에 1~9 모두 넣을 수 없다

이런 경우엔 (1,3)에 다음에 넣을 수 있는 숫자를 대입한 뒤, (1,4)에 1~9 중에 넣을 수 있는 숫자를 다시 검사해야 한다

하지만 하나의 경우만 출력하기 위해 for문이 끝난 뒤, queue에 (1,4)를 넣지 않았다

따라서 (1,3)에 넣을 수 있는 다른 숫자를 찾은 뒤엔 (1,4)가 아니라 (1,6)에 넣을 수 있는 숫자를 찾고 있었다

**결론: 완전탐색 하려면 무조건 처음에 주어졌던 좌표들이 그대로 있어야 함**

    int x;

위 케이스로 확인할 수 있음

solve가 bool 값을 리턴하도록 했고, for문에서 호출한 solve가 true를 반환하면 바로 true를 리턴하도록 변경했음

**결론: 출력은 solve 밖에서 하는게 맞다**

* queue로 풀어도 통과된다. 순서는 상관 없었음
